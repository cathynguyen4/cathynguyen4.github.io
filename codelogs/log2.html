Code Log 2: CSS Measuring Units

Absolute vs. Relative CSS Length Units

When working in CSS, it’s important to understand the difference between absolute and relative units. Absolute units are fixed. That means they do not change based on screen size, parent elements, or user settings. Examples include px and pt. These units try to stay the same size no matter what.

Relative units, on the other hand, are based on something else. They adjust depending on the font size, the root element, or even the size of the browser window. Examples include em, rem, vw, and vh. Relative units are extremely useful for responsive design because they scale naturally across devices. In modern web development, relative units are often preferred because they improve flexibility and accessibility.

1. px and pt

The px unit stands for pixel. It is the most common unit used in web design. Even though pixels seem absolute, they are actually CSS pixels, meaning they are adjusted by the browser to look consistent across different devices. Pixels are good for borders, icons, and small layout details.

The pt unit stands for point. It is mostly used in print design. One point equals 1/72 of an inch. On screens, it is less common and not recommended for responsive layouts.

2. em and rem

The em unit is relative to the font size of its parent element. If a parent has a font size of 20px, then 1em equals 20px. This can sometimes cause compounding effects when elements are nested.

The rem unit stands for "root em." It is relative to the root HTML element’s font size instead of the parent. This makes it more predictable and easier to control in larger layouts. Many developers prefer rem for consistent typography.

3. vw and vh

The vw unit stands for viewport width. One vw equals 1% of the browser window’s width. The vh unit stands for viewport height, and 1vh equals 1% of the browser window’s height.

These units are especially useful for full-screen layouts, hero sections, and responsive typography that scales with screen size. However, they should be used carefully on mobile devices because browser toolbars can affect the viewport height.
