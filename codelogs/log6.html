Codelog 6: References

For this Code Log, I looked up the CSS property display. I chose this property because it is one of the most important parts of CSS layout. The display property controls how an element appears on a page and how it interacts with other elements around it. For example, it can make something act like a block, an inline element, a flex container, or even hide it completely. Since layout is such a big part of web design, I thought this would be a useful topic to compare across different code reference sites.

Resources I Used

  • MDN Web Docs: display
  • W3Schools: CSS display Property
  • CSS-Tricks: display
  • web.dev: Learn CSS Layout

Differences Between the Resources

Each site explained the display property a little differently. MDN gave the most detailed explanation and included a lot of technical information, syntax, and examples. W3Schools was simpler and easier to read quickly, which makes it useful for beginners. CSS-Tricks explained the property in a more conversational way and helped make the layout ideas easier to understand. web.dev focused more on how display connects to modern page layout, especially flexbox and grid. Overall, all four sites were useful, but they each had a different style and level of detail.

What Makes a Good Code Reference Page?

I think a good code reference page should have a clear definition, simple examples, and show how the code works in real situations. It should also include syntax, browser support, and explanations that are not too confusing for beginners. Interactive examples are also really helpful because they let users see what the code actually does. Out of the four resources I used, MDN stood out as the most complete, while W3Schools was probably the easiest to quickly understand. CSS-Tricks was helpful because it explained things in a more casual way.
