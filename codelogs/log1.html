Code Log 1: HTML Semantic Tags

In HTML, tags are used to structure and describe the content of a webpage. These tags fall into two main categories:

  • non-semantic tags
  • semantic tags.

Understanding the difference between the two is important for creating webpages that are accessible, readable, and easy to maintain.

Non-Semantic Tags

Non-semantic tags do not describe the meaning of the content inside them. They are mainly used for grouping elements or applying styles. Common non-semantic tags include:<div> and <span>. When these tags are used alone, browsers and screen readers do not know what role the content plays on the page.

Semantic Tags

Semantic tags clearly describe the purpose of the content they contain. Examples include <header>, <nav>, <main>, <article>, and <footer>. These tags give meaning to the structure of the page, making it easier for both humans and machines to understand.

Benefits of Using Semantic Tags

  • Accessibility: Screen readers use semantic tags to help users navigate a webpage more easily.
  • Search Engine Optimization (SEO): Search engines can better understand and rank content when semantic tags are used.
  • Code Readability: Semantic HTML is easier to read, understand, and maintain for developers.

By using semantic tags whenever possible, developers create webpages that are more meaningful, accessible, and professional. While non-semantic tags still have their place, semantic tags should be the preferred choice for defining the structure of a webpage.
