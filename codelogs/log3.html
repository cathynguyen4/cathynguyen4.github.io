CSS Media Queries

How does a CSS media query work, and why is it useful?

A CSS media query is a rule that lets your website apply different CSS styles depending on the user’s device or screen conditions. The browser checks the media query’s condition (like the screen width, orientation, or user preferences), and if the condition is true, it uses the CSS inside that media query block.

Media queries are useful because they help you build responsive websites that still look good and feel usable on different devices (phones, tablets, laptops, big monitors). Instead of making separate websites, you can keep one website and adjust the layout, font sizes, spacing, and navigation based on the screen or device.

Four media features supported by media queries (and what they do)

1) max-width

max-width targets screens up to a certain width. This is commonly used for “mobile styles,” because it applies when the screen is small enough. For example, you can stack columns into a single column when the screen is 600px or less.

2) min-width

min-width targets screens starting at a certain width and larger. This is often used for “desktop enhancements,” like adding multi-column layouts, increasing spacing, or showing extra navigation options when there’s more room.

3) orientation

orientation checks whether the device is in portrait or landscape mode. This is helpful for tablets and phones where rotating the screen changes the available horizontal space a lot. You can adjust sizes and layout based on how the device is being held.

4) prefers-reduced-motion

prefers-reduced-motion is based on a user accessibility setting in their operating system. If the user prefers less motion (to avoid dizziness or discomfort), you can reduce or remove animations and smooth scrolling. This improves accessibility and shows good design practice.
