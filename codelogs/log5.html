Codelog 5; Images

Images are used in web design to make pages more interesting, informative, and visually appealing. In HTML, images can be added directly with the <img> tag. In CSS, images can also be used as background images for elements like sections, headers, or boxes.

Images in HTML

The <img> tag is used when an image is part of the actual content of the page. For example, a photo, logo, chart, or product image would usually be added with <img>. It appears in the HTML itself and can include alt text for accessibility. 

<img src="flower.jpg" alt="Pink flower" width="300">

Images in CSS

A background image is added with CSS using the background-image property. This is usually better when the image is decorative and not the main content of the page. For example, a banner image or textured section background would often be added with CSS. 

.banner {
  background-image: url("forest.jpg");
  background-size: cover;
  background-position: center;
}

Difference Between <img> and background-image

The <img> tag places an image directly into the page as content, while background-image places an image behind an element as part of its design.

In terms of code, <img> is written in HTML, while background-image is written in CSS.

In terms of functionality and layout, an <img> element takes up space on the page as its own object, while a background image stays inside the box of another element. The <img> tag is also better for meaningful images because it supports alt text. Background images are usually more decorative.

Other CSS Properties for Background Images

Besides background-image, other CSS properties also affect how background images display.

background-size controls the size of the image. For example, cover makes the image fill the whole element.

background-repeat controls whether the image repeats or only shows once. For example, no-repeat prevents tiling.

background-position controls where the image appears inside the element, such as center, top, or bottom. 

.box {
  background-image: url("leaves.jpg");
  background-size: cover;
  background-repeat: no-repeat;
  background-position: center;
}

Dimensions vs. Resolution

Dimensions are the width and height of a bitmap image in pixels, while resolution refers to how much detail the image contains or how densely those pixels are displayed.

Image File Formats

GIF, PNG, JPG/JPEG, WebP, and SVG are all image formats, but they are designed for different uses.

GIF supports simple animation, but it has limited color quality, so it is not the best choice for detailed photos.

PNG supports lossless quality and transparency, which makes it a good choice for logos, graphics, and screenshots.

JPG/JPEG is commonly used for photographs because it compresses images well, but it loses some quality when compressed.

WebP is a modern format that usually creates smaller file sizes while still keeping good quality, so it is often very useful for websites.

SVG is different from the others because it is a vector format instead of a bitmap format. That means it can scale up or down without getting blurry, which makes it ideal for icons, logos, and simple illustrations.

Some formats are better than others depending on the purpose. JPG is usually best for photos, PNG is better for transparent graphics, GIF is useful for simple animations, WebP is great for web performance, and SVG is best for scalable graphics.
